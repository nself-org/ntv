# nTV Backend

Self-hosted media server backend for nTV. Runs on the nSelf CLI stack with the nMedia plugin bundle.

The nTV Flutter app connects to this backend for library management, streaming, EPG scheduling, and metadata. Free IPTV playback (any M3U URL) works without a license. The nMedia bundle ($0.99/mo) adds transcoding, DVR, stream proxying, TMDB metadata, and content acquisition.

---

## Prerequisites

- [nSelf CLI](https://nself.org/docs/getting-started) installed (`nself --version`)
- Docker running locally
- nMedia bundle license key from [nself.org/pricing](https://nself.org/pricing) (for pro media features)
- TMDB API key from [themoviedb.org](https://www.themoviedb.org/settings/api) (free, for metadata)

---

## Quick Start

```bash
# 1. Copy the environment template
cp .env.example .env

# 2. Open .env and set your keys
#    NSELF_PLUGIN_LICENSE_KEY=nself_pro_your_key_here
#    TMDB_API_KEY=your_tmdb_api_key

# 3. Save your license key to the CLI keystore
nself license set nself_pro_your_key_here

# 4. Install plugins, build, and start
make setup && make build && make start
```

The backend is ready when `make status` shows all services healthy.

## Plugins

### nMedia Bundle (pro, $0.99/mo)

| Plugin | What it does |
|--------|--------------|
| `media-processing` | Video transcoding, thumbnail generation, subtitle extraction |
| `streaming` | HLS/DASH adaptive streaming, DVR buffer management |
| `stream-gateway` | IPTV stream proxying, health monitoring, retry logic |
| `epg` | Electronic Programme Guide ingestion and scheduling |
| `tmdb` | Movie and TV metadata, posters, ratings from TMDB |

### Free Plugins (MIT, no license required)

| Plugin | What it does |
|--------|--------------|
| `torrent-manager` | Torrent queue, ratio tracking, watch folder |
| `content-acquisition` | Download coordination, library import, file organization |
| `subtitle-manager` | Subtitle search, download, and sync |

## Service URLs

After `make start`, the backend is available at:

| Service | URL |
|---------|-----|
| Hasura GraphQL | `http://localhost:8080/v1/graphql` |
| Hasura Console | `http://localhost:9695` |
| Auth | `http://localhost:4000` |
| Storage | `http://localhost:8080/v1/storage` |
| MeiliSearch | `http://localhost:7700` |

For production or staging, replace `localhost` with your server domain.

---

## Connecting the nTV App

1. Open nTV on your device.
2. Go to Settings > Backend Configuration.
3. Enter the backend URL: `http://<your-server>:8080/v1/graphql`
4. Enter the auth URL: `http://<your-server>:4000`
5. Save and restart the app.

For local development on the same machine, use `http://localhost:8080/v1/graphql`.

## Common Commands

```bash
make setup      # Install all plugins (run once)
make build      # Regenerate docker-compose from nself.yaml + .env
make start      # Start all services
make stop       # Stop all services
make status     # Check service health
make logs       # Tail all service logs
make db-console # Open database console
```

## Generated Files

`nself build` generates `docker-compose.yml` from your `nself.yaml` and `.env` configuration. This file carries the header:

```
# GENERATED BY nself build — DO NOT HAND EDIT
```

Do not edit `docker-compose.yml` directly. Changes made by hand will be overwritten on the next `nself build`. To adjust service behavior, update `nself.yaml` or `.env` and rebuild.

## Environment Files

| File | Purpose |
|------|---------|
| `.env.example` | Template with all available vars (committed) |
| `.env` | Your local config (gitignored) |
| `.env.dev` | Team dev defaults (shared, committed) |
| `.env.prod` | Production config (shared, committed) |
| `.env.secrets` | Production secrets (gitignored, never committed) |

Later files override earlier ones. `.env` is always highest priority.
