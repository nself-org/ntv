# nTV Backend

Generated by `nself init --full --name ntv --domain ntv.local` (S24-T02, 2026-04-17).

This directory holds the nSelf backend configuration that nTV clients connect to. **Never edit `docker-compose.yml` or `nginx/sites/` by hand** — they're generated from `.env` by `nself build`.

## Quick start

```bash
cd ntv/.backend

# 1. Set your license key (for Pro plugins in the nTV bundle)
nself license set nself_pro_xxxxx...

# 2. Install free nTV plugins
nself plugin install torrent-manager
nself plugin install content-acquisition
nself plugin install subtitle-manager

# 3. Install Pro nTV plugins (requires valid license)
nself plugin install media-processing
nself plugin install file-processing
nself plugin install streaming
nself plugin install stream-gateway
nself plugin install epg
nself plugin install tmdb
nself plugin install recording

# 4. Build and start
nself build
nself start
```

## Required services (enabled in `.env`)

| Service | Why nTV needs it |
|---------|------------------|
| PostgreSQL | Library DB, user prefs, watch history |
| Hasura | GraphQL API for the Flutter client |
| Auth | User accounts + JWT |
| Nginx | SSL termination + routing |
| Redis | Transcoding job queue + cache |
| MinIO | S3-compatible media file storage |
| MeiliSearch | Library full-text search |

## nTV bundle

See `.env` — the `NMEDIA_BUNDLE_ENABLED` flag and the plugin list. Free plugins install without a license. Pro plugins require an active nTV bundle subscription ($0.99/mo) or ɳSelf+ ($49.99/yr). See the marketing site at [ntv.nself.org](https://ntv.nself.org) for details.

## Environment files

- `.env` — active config (never commit, contains generated secrets)
- `.env.dev` / `.env.staging` / `.env.prod` — per-environment overrides
- `.env.secrets` — generated secrets (gitignored)
- `.env.example` — sanitized template
